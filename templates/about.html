Architecture Details

This project demonstrates a fully automated CI/CD pipeline integrated with a dynamic, stateful Flask application.

1. Source Control

GitHub handles version control. The main branch is protected and PRs trigger testing.

2. CI/CD Runner

GitHub Actions runs `pytest` and lints the code. On success, it builds a Docker image and pushes it.

3. Containerization

Docker ensures the environment is identical across develop, test, and production.

4. Data Persistence

SQLite/Volumes handle data durability through container restarts, ensuring deploy logs are maintained.
